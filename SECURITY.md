# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within TRIGRS-PyWrap, please send an email to the maintainer at fjgomezc@eafit.edu.co. All reports will be addressed promptly.

**Please do not report security vulnerabilities through public GitHub issues.**

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Security Best Practices

When using TRIGRS-PyWrap in production environments:

- Always validate input data before processing
- Use virtual environments to isolate dependencies
- Keep dependencies up to date
- Review output data for unexpected values before downstream use
